
# Module Overview — Teaching with FAIROTrials

**Audience:** medical students, MPH students, residents, junior data scientists  
**Duration:** 4 x 60–90 minute sessions

## Learning Objectives
- Explain the FAIR principles and why clinical trial transparency matters.
- Run seven validators and interpret CSV outputs.
- Identify outcome switching and harms‑reporting issues from real publications.
- Prototype a simple validator and write a unit test.
- Reflect on ethics and policy implications of transparency tooling.

## Session Plan
1. **FAIR + Registries 101** — short lecture + demo (see Module_FAIR_and_Transparency.md).
2. **Hands‑on Lab 1–2** — run validators, read CSVs, discuss interpretation.
3. **Hands‑on Lab 3** — add/replace trials in sample JSON, re‑run, compare.
4. **Hands‑on Lab 4** — implement a tiny validator + test; share PRs.

## Assessment
- Short reflection memo (1 page) describing one transparency problem found.
- Pass/fail rubric in `Module_Assessment_Rubrics.md`.
